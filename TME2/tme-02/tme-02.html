Étape 2: les classifieurs¶

Un classifieur (binaire dans ce cours) permet, pour chaque point d'entre donn, de calculer un score (qui est une valeur de $\mathbb{R}$). Si le score est strictement ngatif, la donne est associe l'tiquette $-1$, sinon elle est associe l'tiquette $+1$.

Représentation orientée objets¶

Pour implmenter nos algorithmes d'apprentissage et pour reprsenter les donnes, nous allons utiliser quelques lments de programmation oriente objet (POO) qu'autorise le langage Python.

Si vous ne connaissez rien la POO, il est fortement conseill de vous familiariser avec elle, par exemple en commenant par lire une documentation ou suivre un tutorial sur le web (par exemple, http://apprendre-python.com/page-apprendre-programmation-orientee-objet-poo-classes-python-cours-debutants).

La classe Classifier¶

Les classifieurs que l'on va implmenter seront dfinis l'aide d'une classe qui va driver de la classe Classifier suivante. Cette classe est une classe "abstraite", elle sera drive par hritage pour dfinir des classifieurs particuliers (knn, arbres de dcision, etc.).

Cette classe contient 5 mthodes:

  • __init__: qui permet d'initialiser les paramtres du classifieur. Les classes filles tendront cette mthode en rajoutant les paramtres qui les concernent. Le paramtre obligatoire d'un classifieur est le nombre de dimensions des donnes qu'il aura traiter.
  • train: c'est la mthodes qui permet d'entraner le modle, on doit donc donner en argument le dataset qui sert l'entranement.
  • score: cette mthode est utilise une fois le classifieur entrain. Elle prend en argument la description d'un exemple et elle rend un score (une valeur de $\mathbb{R}$).
  • predict: cette mthode, comme la prcdente, s'utilise une fois le classifieur entran. Elle rend $\hat y$, le label (-1 ou +1) qui est prdit par le classifieur pour la description de l'exemple donne en argument. Gnralement, cette valeur de prdiction $\hat y$ s'obtient partir du score: si le score est strictement ngatif $\hat y$ vaut $-1$, sinon il vaut $+1$.
  • accuracy: comme les 2 prcdentes, cette mthode s'utilise une fois le classifieur entran. Elle permet de calculer la qualit du classifieur sur un dataset donn, cette qualit est mesure par le taux de bonne classification du classifieur sur le dataset. C'est donc une valeur de $[0,1]$ qui s'obtient divisant le nombre d'exemples du dataset qui sont bien classs par le classifieur par le nombre total d'exemples du dataset.

Les mthodes train, score et predict ne seront pas dfinies dans la classe Classifier mais dans les classes qui hriteront ce cette classe car ces mthodes dpendent du modle que l'on choisira d'implmenter.

Par contre, la fonction accuracy peut tre dfinie ds maintenant dans cette classe Classifier.

les $k$-plus proches voisins ($k$-ppv ou $k$-NN)¶

Le classifieur des $k$-plus proches voisins est trs intuitif: il s'agit de prdire la classe majoritaire dans le voisinage d'un point en examinant ses $k$ plus proches voisins (en anglais: $k$ nearest neighbors). La figure ci-dessous illustre bien le principe gnral de l'algorithme:

Cet algorithme d'apprentissage est particulier: il n'y a pas d'apprentissage, il faut juste stocker tous les points d'apprentissage pour les rutiliser plus tard !! En contrepartie, l'infrence coute trs cher: il faut analyser tout le dataset pour chaque prdiction afin de trouver les $k$ plus proches voisins de l'exemple prdire.

Description de la classe ClassifierKNN (qui tend Classifier):

  • dans le constructeur, on prcise, en plus de la dimension du dataset, le nombre de voisins utiliser (valeur de $k$)
  • dans la mthode train, on donne le dataset qui doit servir de dataset de rfrence et qui va donc tre stock
  • dans la mthode score, pour une description d'exemple $x$ donne :
    • 1) on construit le tableau des distances entre $x$ et les exemples du dataset de rfrence,
    • 2) on trie ce tableau grce la mthode numpy.argsort,
    • 3) le score qui doit tre rendu est alors la proportion de classe +1 parmi les $k$ plus proches voisins de $x$. Ce score est donc un nombre rel de $[0,1]$ qui vaut $0.0$ si tous les $k$ voisins de $x$ sont de la classe $-1$ et qui vaut $1.0$ si tous les $k$ voisins de $x$ sont de laclasse $+1$.
      Par consquent, si $k$ vaut $1$ (on ne regarde que le plus proche voisin), la mthode score rend donc soit $0.0$ soit $1.0$ selon la classe de ce voisin.
  • dans la mthode predict, pour une description d'exemple $x$ donne, on utilise le score rendu par la fonction prcdente pour rendre la classe de l'exemple, c'est--dire soit $-1$ soit $+1$.
